遊戲策劃案：《創世者：白紙宇宙》 (Genesis: The Blank Canvas)
一、 遊戲概述 (Game Overview)
遊戲類型： AIGC 沙盒建造 / 大型多人在線虛擬社交 (MMO-UGC)
平台： iOS / Android (優先)，後續登陸 PC
美術風格： 初始為極簡純白網格風格，隨著玩家創造，逐漸變為Low-Poly（低多邊形）或高質量 2.5D 手繪風格。
核心願景： 給玩家一個純白的無限宇宙，讓他們用「語言」作為魔法，創造萬物，並建立起由玩家主導的龐大自由經濟體系。
二、 世界觀與初始體驗 (World & Onboarding)
背景設定： 宇宙重置，一切歸零。玩家扮演被選中的「言靈法師（Lexicon）」，降生在一個無限延伸的純白色網格空間（The White Room）。
初始體驗：
玩家登入遊戲，周圍只有白色的地面和無盡的空間。
系統引導（NPC）：「世界空無一物，但你的語言擁有重量。試著說點什麼。」
玩家在引導下輸入「一顆綠色的蘋果樹」。幾秒後，光芒閃爍，一顆由 AI 生成的蘋果樹實實在在地出現在純白空間中。
玩家擁有了自己的第一塊「領地（10x10格）」，開始了創世之旅。
三、 核心玩法：創造、建設與探索
1. 創造系統 (The Creation Engine)
為了解決你擔心的「提示詞過長導致 API 成本失控」問題，創造過程採用**「配方式」輸入**：
介面設計： 玩家點擊「創造」按鈕，出現一個鍊金術般的UI。玩家需要填入 3-4 個插槽：
[材質] (如：木頭、星空、生鏽的金屬)
[風格] (如：賽博龐克、中世紀、可愛風)
[物品] (如：椅子、雕像、飛船)
[自定義細節] (限制 20 字以內的自由文本，如：上面有一隻熟睡的貓)
造物消耗： 點擊生成時，系統會計算複雜度，並提示：「本次造物將消耗 35 點【創世塵埃（Mana）】」。
讀條特效： 生成期間（約需10-15秒），遊戲內會出現一個「魔法陣正在編織」的酷炫動畫，掩蓋 AI 生成的延遲。
2. 領地與世界擴張
個人島嶼： 每個玩家初始擁有一塊專屬領地。玩家可以將創造出的物品自由擺放、旋轉、搭建（類似《模擬市民》或《動物森友會》的裝修模式）。
無限拼接： 玩家可以走出自己的領地，來到「無盡荒野」。玩家可以花費遊戲幣購買相鄰的白色地塊，擴張自己的家園。最終，無數玩家的領地會接壤，形成巨大的玩家城市。
四、 經濟與交易系統 (核心商業化設計)
這款遊戲有兩種貨幣，這是為了避開法律風險（真金交易/洗錢）並確保你賺錢的設計：
1. 雙代幣系統
【創世塵埃 (Mana)】- 氪金貨幣 (你的利潤來源)
來源： 只能通過 App Store / Google Play 充值購買（例如 $0.99 = 1000 點），或者每日系統低保贈送少量。
用途： 唯一用途是驅動 AI 進行生成。
【星幣 (StarCoin)】- 遊戲內流通貨幣
來源： 遊戲內活動獲得，或賣東西給其他玩家獲得。
用途： 向其他玩家購買物品、購買更多土地、購買角色服裝。
2. 交易市場 (The Bazaar)
這就是你最初設想的「玩家賣桌子、房子」的系統。
實物交易： 玩家 A 花費 Mana 創造了一把「超帥的冰霜龍王劍」（擺設品）。他可以將這把劍掛在拍賣行，定價 500 星幣。玩家 B 覺得太酷了，用 500 星幣買下。
藍圖專利交易 (進階玩法)： 如果玩家 A 創造了一個極具美感的別墅，他可以將其註冊為「專利藍圖」。其他玩家花費 50 星幣購買藍圖，然後自己再掏 Mana 生成出來。玩家 A 享受源源不斷的版權收入。
開發者抽成（你的稅收）： 玩家之間的所有星幣交易，系統（也就是你）自動抽成 10% 的交易稅。這可以回收遊戲內的貨幣，控制通貨膨脹，維持經濟健康。
五、 社交與留存機制 (如何讓玩家一直玩？)
如果只是單純的建造，玩家很快會無聊。必須加入社交評比：
世界奇觀排行榜 (The Wonder Board)：
每週舉辦主題建築大賽（例如這週主題是「深海龍宮」）。
玩家參賽，全服玩家點讚投票。獲勝者的作品將被展示在遊戲登入的公共廣場中央，並獲得絕版稱號。
串門與互動：
玩家可以拜訪別人的領地。
如果看到別人領地裡有一張椅子，點擊可以查看作者是誰，並選擇「打賞」或者「一鍵購買同款」（直接消耗自己的 Mana 複製一個，作者獲得分成）。
收集圖鑒：
設立「萬物圖鑒」。玩家每創造出一種全新類別的物品，就能點亮圖鑒，獲得星幣獎勵。滿足玩家的收集癖。
六、 商業化與盈利點總結 (Developer Revenue)
作為開發者，你將通過以下方式合法、合規地賺取豐厚利潤：
API 差價 (核心收入)： 玩家充值買 Mana。你將 Mana 的價格設定為 AI 大模型實際 API 成本的 3~5 倍。玩家創造得越多，你賺得越多。加上你運用了「緩存技術」（相似物品不調用 API 直接發放），這部分利潤率會極高。
通行證 (Battle Pass)： 每月推出 9.99 美金的「創世者通行證」。解鎖特殊的[材質]詞條（如黃金、全息投影）、專屬的角色走路特效、以及每日額外的 Mana 補給。
地塊租售： 當玩家想要在世界最中心（流量最大，最容易賣出東西的地方）購買地塊時，需要消耗大量的資源，促使他們活躍或消費。
七、 開發階段規劃 (Roadmap)
Phase 1：核心驗證期 (MVP) - 第 1~2 個月
完成 2D 等距視角或簡單 3D 的渲染。
接通 OpenAPI Moderation（敏感詞過濾）和 Meshy/Luma AI 的生成 API。
實現「輸入配方 -> 扣除 Mana -> 出現物品」的核心單機循環。
Phase 2：經濟與後端建立 - 第 3~4 個月
接入蘋果/谷歌內購系統。
建立後端緩存庫（相似提示詞直接發放現成模型）。
開發交易行系統，實現玩家間物品買賣。
Phase 3：聯機與上架 - 第 5~6 個月
接入 Photon/Nakama 實現多人同屏看見彼此。
加入好友系統和拜訪系統。
準備宣傳物料，主打「全網第一款 AI 驅動的 MMO 沙盒」，正式雙端上架。
